一、类加载器

1.1 类的加载时机

当我们第一次使用某个类时，这个类会加载到方法区。

  • 创建类的实例(创建对象)
  • 类的静态变量，或者为静态变量赋值
  • 类的静态方法
  • 使用反射方式来强制创建某个类或接口对象的java.lang.Class对象。
  • 初始化某个类的子类
  • 直接使用java.exe命令来运行某个主类

以上六种情况的任何一种，都可以导致JVM将一个类加载到方法区。

NIO

NIO介绍

同步和异步（线程通信的机制）

  • 同步：线程在完成某个功能时，必须等待这个功能执行完才能再继续往下执行。
  • 异步：线程在完成功能的时候，不用等到该功能执行结束也可以去做其他事情。 得到该功能结束后，系统会通知线程该功能执行完了

阻塞和非阻塞(线程的状态)

  • 阻塞：线程在执行任务时，会挂起。
  • 非阻塞：线程在执行任务时，不会挂起，可以继续执行其他任务。

一、File类

java.util.File类是文件和目录路径名的抽象表示，主要用于文件和目录的创建、查找和删除等操作。

1.1 构造方法

File类的构造方法

  • File(String pathname)：根据文件或文件夹得路径创建File对象
  • File(String parent, String child)：根据父路径和子路径创建一个File对象。
    • 文件的路径：E:\file_test\aa.txt
    • 父路径：E:\file_test
    • 子路径：aa.txt
  • File(File parent, String child)：根据父路径和子路径创建一个File对象

小贴士：

  • File可以表示计算机中的文件或文件夹

  • File表示的文件或文件夹可以存在，也可以不存在。
