一、数据库介绍

1.1 什么是数据库

数据库：存储数据的仓库,本质上就是存储数据的文件系统，方便我们管理数据。

1.2 数据库管理系统层次

数据库管理系统（Database Management System[DBMS]）：指一种操作和管理数据库的大型软件。

MySQL软件–>仓库—>表–>记录（数据）

1.4 常见的关系型数据库

  • MYSQL：开源免费的数据库，小型的数据库.已经被Oracle收购了.MySQL6.x（商业、社区）版本也开始收费。

三、SQL

3.1 概述

SQL：结构化查询语言（Structured Query Language）

  • 通过sql语句，可以实现对数据库的增删改查
  • CRUD：create 创建、 retrieve(read) 检索、update 更新、delete 删除

3.2 SQL方言

SQL是一套标准，所有的数据库厂商都实现了此标准；但是各自厂商在此标准上增加了特有的语句，这部分内容我们称为方言。

方言举例：

  • MySQL注释 #
  • MySQL分页 limit

3.3 SQL书写规范

  • SQL语句可以单行或多行书写，最后以分号结束
  • SQL语句在windows平台下不区分大小写，关键字大写
  • SQL的注释：
    • 单行注释：-- 注释内容
    • 多行注释：/* 注释内容 */

3.4 SQL分类

  • DDL：(Data Definition Language)数据定义语言
    • 用来定义数据库对象：数据库，表，列等。关键字：create,drop,alter等
  • DML：(Data Manipulation Language)数据操作语言
    • 用来对数据库中表的数据进行增删改。关键字：insert,delete, update等
  • DQL：(Data Query Language) 数据查询语言
    • 用来查询数据库中表的记录(数据)。关键字：select, where等
  • DCL：(Data Control Language)数据控制语言
    • 用来定义数据库的访问权限和安全级别，及创建用户。关键字：grant,revoke等
  • TCL：(Transaction Control Language) 事务控制语言
    • 用于控制数据库的事务操作，关键字; commit,savepoint,rollback等

六、数据库约束

6.1 概述

作用

对表中的数据进行限定，保证数据的正确性、有效性和完整性。

分类

  1. 主键约束：表示当前记录的（唯一+非空）标识。类似于人类的身份证号
  2. 唯一约束：表示当前记录的唯一性
  3. 非空约束：表示当前记录不能为空
  4. 默认值：我们可以为当前字段指定默认值，如果未指定，默认值就为null
  5. 外键约束：限定二张表数据的正确性、有效性和完整性

6.2 实现

6.2.1 主键约束

作用：限定某一列的值非空且唯一， 主键就是表中记录的唯一标识。通常使用id作为主键

一张表只能有一个主键，但是一个主键可以包含多个字段

方式一：在建表的时候,在字段名后面通过primary key 声明

方式二：在建表的时候,在约束区域通过primary key 声明

方式三：通过修改表结构,添加主键约束

小贴士：

  • 如果要设置主键的字段存在重复数据，将设置失败…
扩展

主键自增：要求主键类型支持自增，一张表只能指定一个自增器

truncate：摧毁表，重构表

truncate table 表名 和 delete from 表名 区别是什么?

  • DML分类:delete from，相当于橡皮擦，一条条记录抹除，保留痕迹

  • DDL分类：truncate table，相当于摧毁表，重构表，不保留痕迹，数据量大，性能更好

6.2.2 唯一约束

作用：限定某一列的值不能重复

小贴士

  • 唯一约束，可以出现多个null…

6.2.3 非空约束

作用：限定某一列的值不能为null

小贴士：

  • 虽然给 name字段设置了 二个约束（unique + not null），功能类似，但它并不是主键约束

6.2.4 默认值

作用：限定某一列的默认值，在没有指定的情况下所有列的默认值为null
