    1.3 外键约束

    功能：限定二张表有关系的数据，保证数据的正确性、有效性和完整性

    外键约束相关语法：（在从表中设置）

    • 创建表指定外键字段
    • 修改表指定外键
    • 删除外键约束

    外键约束的特点：

    • 主表不能删除从表已引用的数据
    • 从表不能添加主表未拥有的数据
    • 先删除从表数据再删除主表数据
    • 先添加主表数据在添加从表数据
    • 外键字段可以为空但不能非法关联

    小贴士：

    • 外键约束虽然功能强大，但是在互联网项目中不会使用，我们可以通过后台代码实现数据的校验…

    4.5 事务特性[ACID]

    原子性：atomicity

    • 如果一个包含多个步骤的业务操作，被事务管理，那么这些操作要么同时成功，要么同时失败。

    一致性：consistency

    • 事务执行前后，保证数据完整性(数据总量)一致

    隔离性：isolation【并发】

    • 多个事务之间要求隔离，互不干扰

    持久性：durability

    • 事务一旦成功提交，持久化到磁盘，即使数据库出现问题也不会对数据不会产生影响

    4.6 事务隔离性[并发]

    多个事务之间隔离的，相互独立的。但是如果多个事务操作同一批数据，则会引发一些问题，设置不同的隔离级别就可以解决这些问题。

    4.6.1 MySQL数据库隔离级别

    级别 名字 隔离级别 脏读 不可重复读 幻读 数据库默认隔离级别
    1 读未提交 read uncommitted 是 是 是
    2 读已提交 read committed 否 是 是 Oracle和SQL Server
    3 可重复读 repeatable read 否 否 是 MySQL
    4 串行化 serializable 否 否 否

    设置数据库的隔离级别的不同考虑，选择不同的隔离级别：

    • 考虑性能： 1 > 2 > 3 > 4

    • 考虑安全：4 > 3 > 2 > 1

    • 综合考虑：2 、3
