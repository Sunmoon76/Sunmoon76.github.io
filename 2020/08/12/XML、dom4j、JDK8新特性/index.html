一、XML

1.1 xml介绍

Extensible markup language：可扩展标记语言

  • 可扩展：标签都是自定义的

作用：存储数据，传输数据

1.2 xml语法

1.2.1 文档声明

格式：

  • <?xml version="1.0" encoding="UTF-8"?>

注意：

  • 必须以 <?xml 开头，以 ?> 结尾
  • 文档声明必须放到xml的第一行
  • 文档声明可以省略
  • 文档声明有两个属性
    • version:表示版本号
    • encoding:表示编码

1.2.2 元素

格式：

  • <元素名></元素名>
  • <开始标签></结束标签>

注意：

  • 元素一般都是由开始标签、结束标签、、元素的内容(元素体)组成的。
  • 元素体可以是文本内容，也可以是其他标签
  • 每一个xml有且仅有一个根标签
  • 空元素，如果某个元素没有元素体，它就是一个空元素。空元素可以写成自闭和的形式，格式：<元素名/>
  • xml标签区分大小写
  • 在xml中，元素命名不要使用xml,Xml,XML这些名字，也不要带有一些特殊字符。

1.2.3 属性

格式：

  • <开始标签 属性名1=属性值1 属性名2=属性值2 ...></结束标签>

注意事项：

  • 属性要写在开始标签中。
  • 属性可以有任意个，多个属性之间用空格隔开，一个元素中的多个属性，属性名不能相同
  • 属性值必须用双引号或单引号包裹，推荐双引号
  • 属性名在命名时不要有特殊字符，属性名不能以数字开头

1.2.4 注释

格式：

  • <!-- 注释内容 -->

注意：

  • 在xml中，注释也是不会被解析的。

1.3 xml约束

概念：规定xml文档的书写规则

  • 作为框架的使用者(程序员)：
    1. 能够在xml中引入约束文档
    2. 能够简单的读懂约束文档

xml的写法是非常灵活的，但是很多情况下太灵活也不是一件好事，所以我们要给xml定义一个规范，规范xml中应该写什么内容。这个规范就是约束。

约束可以限制元素名，属性，元素体，元素的顺序等等

xml约束有两种：dtd，schema

1.3.1 DTD约束

如果要使用约束，需要把约束给引入过来。这个引入也称为导入。

dtd的三种引入方式

  • 内部DTD，将dtd嵌入到xml文件中，支队当前的xml文件有效

  • 外部dtd - 本地dtd：将约束的规则定义在外部dtd文件中

  • 外部dtd - 公共dtd：将dtd文件放在网络中。

1.3.2 schema

引入schema约束的方式和引入dtd约束的方式一样，都是从约束文件的注释中找到相对应得代码给复制到xml文件中。
复制过来得schema注释中的代码是一个开始标签，需要我们手动补齐结束标签

  • 引入过来的标签其实就是一个根标签

解析：

xmlns="http://www.xxx.cn":表示引入过来的schema文件的名称空间

xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance":引入的是w3c官方的内容，固定写法。

xsi:schemaLocation="http://www.xxx.cn bookshelf.xsd": 引入的schema文件的位置。

  • 格式：名称空间 位置

名称空间：

  • 隐式名称空间（之前用的都是隐式名称空间）
  • 显式的名称空间（给名称空间起一个别名，通过别名去使用元素）
    • xmlns:别名 = “名称空间”
    • 使用方式：
      • <别名:元素名>

二、dom4j

2.1 XML解析

XML解析就是获取xml文件的内容。

常见的解析方式：

  • DOM解析(会把整个xml文件都加载到内存中，然后进行解析)
    • 优点:会保留整个xml文档的结构，可以对里面的元素进行增删操作。
    • 缺点：有可能会内存溢出
  • SAX解析(会逐行解析xml，没解析一行，就释放一行)
    • 优点：效率高，不会内存溢出
    • 缺点：不能保留xml文档结构，只能读
  • PULL解析(安卓内置的，类似SAX)

常见的解析工具包：

  • JAXP：sun公司提供支持DOM和SAX开发包

  • Dom4j：比较简单的的解析开发包(常用)

  • JDom：与Dom4j类似

  • Jsoup：功能强大DOM方式的XML解析开发包，尤其对HTML解析更加方便

2.2 dom4j的基本使用

2.2.1 dom4j解析原理(dom解析)：

  • dom解析会将整个xml文件读取到内存中，会生成一个DOM树(在xml中，每一个元素，属性，文本都是DOM树中的节点)，并获 得一个Document对象，通过Document对象就可以对DOM进行操作。

2.2.2 使用dom4j解析xml

解析相关的API

  • SAXReader：核心类

    • Document read(String systemId)：read方法可以读取xml文件，生成dom树创建Document对象

  • Document:表示整个xml文档

    • Element getRootElement();：表示获取根元素

  • Element：表示元素

    • List elements([String ele] )： 获得指定名称的所有子元素。可以不指定名称
    • Element element([String ele])：获得指定名称第一个子元素。可以不指定名称
    • String getName()：获得当前元素的元素名
    • String attributeValue(String attrName)：获得指定属性名的属性值
    • String elementText(Sting ele) ：获得指定名称子元素的文本值
    • String getText()：获得当前元素的文本内容

使用步骤：

  • 准备工作(导包、准备要解析的xml文件)
  • 开始写代码，创建SAXReader对象
  • 调用read方法，读取xml，得到Document对象
  • 调用getRootElement获取根元素
  • 开始解析操作

2.3 XPath

XPath可以根据一个路径表达式获取xml中的指定元素。

XPath使用步骤：

  • 导入jar包(dom4j-1.6.1.jar 和 jaxen-1.1-beta-6.jar)
  • 通过dom4j的SAXReader读取xml文件，获取Document对象
  • 调用方法，根据XPath表达式获取元素
    • document根据XPath表达式获取元素的方法：
      • List selectNodes("表达式")：获取多个满足要求的元素
      • Node selectSingNode("表达式")：获取第一个满足要求的元素

表达式规则：

  • 绝对路径表达式方式 例如: /元素/子元素/子子元素…
  • 相对路径表达式方式 例如: 子元素/子子元素.. 或者 ./子元素/子子元素..
  • 全文搜索路径表达式方式 例如: //子元素//子子元素
  • 谓语（条件筛选）方式 例如: //元素[@属性名=属性值]

3.3 Base64

3.3.1 概述

Base64是网络上最常见的用于传输8Bit字节码的编码方式之一，Base64就是一种基于64个可打印字符
来表示二进制数据的方法。

在Java 8中，Base64编码已经成为Java类库的标准。

Java 8 内置了 Base64 编码的编码器和解码器。

Base64工具类提供了一套静态方法获取下面三种Base64编解码器：

  • 基本：输出被映射到一组字符A-Za-z0-9+/，编码不添加任何行标，输出的解码仅支持A-Za-z0-
    9+/。

  • URL：输出映射到一组字符A-Za-z0-9+_，输出是URL和文件。

  • MIME：输出隐射到MIME友好格式。输出每行不超过76字符，并且使用’\r’并跟随’\n’作为分割。
    编码输出最后没有行分割。

Base64用来编码和解码

3.3.2 Base64的内部类和方法

内部类：

  • static class Base64.Decoder ：用来解码
  • static class Base64.Encoder ：用来编码

获取编码器、解解器方法：

  • static Base64.Decoder getDecoder()：获取一个标准的Base64解码器
  • static Base64.Decoder getEncoder()：获取一个标准的Base64编码器
  • static Base64.Decoder getUrlDecoder()：获取一个标准的URL解码器
  • static Base64.Decoder getUrlEncoder()：获取一个标准的URL编码器
  • static Base64.Decoder getMimeDecoder()：获取一个标准的MIME的解码器
  • static Base64.Decoder getMimeEncoder()：获取一个标准的MIME的编码器

Base64.Encoder方法：

  • String encodeToString(byte[] src)：编码，将byte数组编码成字符串

Base64.Decoder方法：

  • byte[] decode(String src)：解码，将字符串编程byte数组
