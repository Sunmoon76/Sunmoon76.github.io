二、Properties类(属性集)

Properties是一个双列集合，也叫做属性集。

特点：

  • Properties实现类Map接口，拥有Map接口中所有的方法
  • Properties没有泛型，键和值都是字符串
  • Properties可以从流中(文件中)中读取键值对

构造方法：

  • Properties():空参的构造方法

其他方法：

  • Object setProperty(String key, String value):添加键值对
  • String getProperty(String key):根据键获取值
  • Set<String> stringPropertyNames()：获取所有键并放入Set集合中返回

2.1 Properties有关流的方法

Properties有关流的方法，可以从文件中获取键值对：

  • void load(InputStream inStream)：参数需要传递字节输入流
  • void load(Reader reader)：参数需要传递字符输入流

load方法可以传递输入流，该输入流绑定的文件是什么，就可以从该文件中加载(读取)键值对

使用load方法读取文件的要求：

  • 文件一般以【.properties】结尾(软性要求，properties文件也叫做配置文件)
  • 文件中的键值对必须按照指定格式存放(硬性要求)
    • 例如：
      • 键 = 值(username = jack)
      • 键和值之间用=号隔开，每个键值对换一行

三、ResourceBundle

除了Properties之外，在Java中还提供了ResourceBundle用来读取properties配置文件中的内容。

ResourceBundle操作更简便。

ResourceBundle是一个抽象类，如果要用，需要使用子类，他的子类对象不是需要我们new的，而是要通过ResourceBundle的静态方法获取的。

ResourceBundle获取对象的方法：

  • static ResourceBundle getBundle(String baseName)：获取ResourceBundle子类对象，参数表示要操作的配置文件

ResourceBundle对配置文件位置的要求：

  • ResourceBundle读取的properties文件必须放在src目录下，在调用getBundle传递文件路径时，不要带扩展名。

ResourceBundle其他方法：

  • String getString(Stirring key)：根据键获取值。

四、缓冲流

4.1 概述

缓冲流,也叫高效流，是对4个基本的FileXxx流的增强，所以也是4个流，按照数据类型分类：

  • 字节缓冲流
    • 字节输入缓冲流：BufferedInputStream
    • 字节输出缓冲流：BufferedOutputStream
  • 字符缓冲流
    • 字符输入缓冲流：BufferedReader
    • 字符输出缓冲流：BufferedWriter

小贴士：

  • 缓冲流的特点是效率高，缓冲流内部有一个缓冲区，可以提高效率【缓冲区使用的就是数组】
  • 缓冲流本身并不具备读或写的功能，它的作用是给其他流提供加速

4.2 字节缓冲流

构造方法：

  • BufferedInputStream(InputStream in)：参数需要传递一个字节输入流
  • BufferedOutputStream(OutputStream out)：参数需要传递一个字节输出流

字节缓冲流的使用方式、方法和字节流的方法相同。

效率对比及使用示例：

对比上面三种方式复制文件，可以得知，效率是越来愈高的，使用缓冲流进行文件的操作可以很大程度的提高效率。

4.3 字符缓冲流

构造方法：

  • BufferedReader(Reader in)：参数需要传递字符输入流
  • BufferedWriter(Writer out)：参数需要传递字符输出流

字节缓冲流的使用方式、方法和字节流的方法相同。

字符缓冲流中的特有方法：

  • 在BufferedWriter里面有一个方法newLine(),可以实现跨平台的换行
    • void newLine():写一个换行，这个换行是跨平台的
  • 在BufferedReader里面有一个方法readLine()，可以读一行数据
    • String readLine()：一次读取一行数据，并以字符串形式返回读取到的数据。如果已经读取结束了，返回值是null

五、转换流

5.1 字符编码和字符集

编码和解码：

  • 编码：字符 -> 字节
  • 解码；字节 -> 字符

字符编码Character Encoding : 就是一套自然语言的字符与二进制数之间的对应规则。

字符集Charset：也叫编码表。是一个系统支持的所有字符的集合，包括各国家文字、标点符号、图形符号、数字等。

字符集 占用字节数 描述
ASCII码表 1字节 有128个英文字母数字以及标点符号。
ISO8859-1(Latin) 1字节 有256个内容。包含了ASCII码表的内容以及拉丁文。
GB2312(1980) 1或2字节 支持六千多个汉字或者字符
BIG5 1或2字节 支持繁体字
GBK 1或2字节 支持两万多汉字或者字符。

国际标准码表(unicode)：unicode中可以保存任何国家的任何语言，在unicode中每个字符都是使用两个字节保存的。在unicode8.0版本，甚至保存emoji。

万国码：UTF-8

  • UTF-32：每一个字符都使用4个字节保存。
  • UTF-16：每一个字符都使用2或3或4个字节保存。
  • UTF-8：每一个字符都使用1或2或3或4个字节保存。

在GBK中每一个汉字占2个字节。

在UTF-8编码中每一个汉字占3个字节。

小贴士：

  • 在IDEA中，使用的默认编码是UTF-8
  • 在windows中， 使用的默认编码是UTF-8（最近win10更新，新版本的win10默认编码已经变成utf-8）

InputStreamReader是使用方式和字符流的使用方式一样

六、序列化

6.1 概述

序列化：把对象信息持久化存储到硬盘中，保存为一个文件。

反序列化：读取保存在硬盘中的文件，进行反序列化，获取对象的信息。

6.2 ObjectOutputStream类

ObjectOutputStream是序列化流，可以将Java程序中的对象序列化到文件中。

构造方法：

  • ObjectOutputStream(OutputStream out)：参数需要传递字节输出流

ObjectOutputStream写对象的方法【特有方法】

  • void writeObject(Object obj)：向文件中写对象

使用示例：

注意!!!

  • 想要进行序列化的接口必须实现Serializable接口

Serializable：序列化接口

  • Serializable里面没有任何东西，这个接口只是起到一个标记作用，必须实现接口，那么类的对象才可以被序列化，才可以写到文件中

6.3 ObjectInputStream类

ObjectInputStream是反序列化流，可以将文件中的对象读取到Java程序中

构造方法：

  • ObjectInputStream(InputStream in)：参数传递一个字节输入流

ObjectInputStream读对象的方法【特有方法】

  • Object readObject():从文件中读取对象

使用示例

注意：

如果使用反序列化流读取数据，如果读取的对象所属的类不存在，那么会报错[ClassNotFoundException]

6.4 序列化中注意事项

static和transient关键字

  • 被static修饰的属性不能被序列化。被static修饰的属性属于类，不属于对象，而序列化操作写的是对象。
  • 如果我们不希望某个属性被序列化，同时不希望使用static关键字，那么可以使用transient。
    • transient表示瞬态，被transient修饰的属性无法被序列化。

例如：

  • private transient int age;

  • private static int age;

6.5 序列化中的序列号

问题：在进行反序列的的时候，从文件中读取对象时，会对比文件中保存的版本号和class文件中的版本号是否一致，如果不一致，就会报错。如果要解决版本号冲突问题，我们可以给类固定一个版本号，不管该类怎么修改，版本号都不变。这样就解决了问题。

解决方法：可以在类中提供一个常量serialVersionUID，该常量就表示类的版本号，我们可以自己手动提供版本号。

定义serialVersionUID要求：

  • 必须被private static final 修饰
  • 必须是long类型的
  • 名字必须叫做serialVersionUID

七、打印流

PrintStream是打印流。

打印流特点：

  • 只有输出，没有输入
  • 打印流写数据十分方便

构造方法：

特有方法：

  • void print(任何类型):写数据
  • void println(任何类型):写数据并换行

八、装饰设计模式

装饰者设计模式：在不改变原有类，并且不适用继承的前提下，对一个类进行增强。

  • 装饰的类和被装饰的类要实现一个接口
  • 在装饰的类中接收被装饰的对象
  • 在需要增强的方法中直接写增强后的内容
  • 如果某个方法不需要增强，直接通过被装饰的类调用该方法

九、commons-io工具包

commons-io是由第三方(Apache)提供的对于IO流操作的工具包

如果要使用第三方工具包，一般需要导入jar包

jar就是java的压缩包，里面会有很多的class文件，供我们使用

如果导入了jar包，这个jar包中的类就都可以使用了

导入jar步骤：

  • 在当前模块下面新建一个文件夹lib
  • 将jar包复制到lib文件夹下
  • 模块上右键，点击 [add as library]
