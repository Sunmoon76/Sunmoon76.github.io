一、File类

java.util.File类是文件和目录路径名的抽象表示，主要用于文件和目录的创建、查找和删除等操作。

1.1 构造方法

File类的构造方法

  • File(String pathname)：根据文件或文件夹得路径创建File对象
  • File(String parent, String child)：根据父路径和子路径创建一个File对象。
    • 文件的路径：E:\file_test\aa.txt
    • 父路径：E:\file_test
    • 子路径：aa.txt
  • File(File parent, String child)：根据父路径和子路径创建一个File对象

小贴士：

  • File可以表示计算机中的文件或文件夹

  • File表示的文件或文件夹可以存在，也可以不存在。

使用示例：

小贴士：

  • 直接输出对象名相当于调用对象的getPath()方法

1.2 常用方法

1.2.1 获取方法

  • String getAbsolutePath()：获取File对象表示的文件或文件夹的绝对路径
  • String getPath()：获取对象的路径名称，以字符串形式返回（创建File对象传递的路径字符串是什么，获取到的就是什么）
  • String getName()：获取文件或文件夹的名字
  • long length()：获取文件字节数大小。只能对文件使用，如果对文件夹使用，得到的是一个不确定的数字
绝对路径和相对路径
  • 绝对路径：是一个完整的路径，从盘符开始的路径。例如：D：\aa.txt
  • 相对路径：是一个简单的路径，不是从盘符开始的，在idea中，相对路径相对的是当前项目，指的是当前项目下的东西。

如果要区分相对路径和绝对路径，只看是不是从盘符开始就可以。

1.2.2 判断功能的方法

  • public boolean exists()：判断File表示的文件或文件夹是否存在
  • public boolean isDirectory()：判断是否是文件夹
  • public boolean isFile()：判断是否为文件

1.2.3 创建、删除功能的方法

  • public boolean createNewFile()：创建文件，如果已经存在返回false

  • public boolean delete()：删除文件或目录

  • public boolean mkdir()：创建文件夹，如果文件夹已经存在，返回false

  • public boolean mkdirs()：创建文件夹，如果文件夹已经存在，那么创建失败【如果上级目录不存在，同时回创建上级目录】

1.3 目录的遍历

  • public String[] list()：获取指定目录下所有文件和文件夹的名字，放入到字符串数组中返回。
  • public File[] listFiles()：获取指定目录下所有文件和文件夹并放入File数组中返回。

注意：

  • 如果File对象表示的是一个文件，那么调用listFiles得到的是null值
  • 如果File的对象表示的目录不存在，调用listFiles得到的也是null值

三、IO概述

Java中的IO操作主要使用java.io包下的内容，进行输入、输出操作，输入也叫做读取数据，输出也叫做写出数据。

3.1 IO的分类

根据数据的流向分为：输入流和输出流。

  • 输入流：把数据从其他设备中读取到内存中的流
  • 输出流：把数据从内存写出到其他设备上的流

根据数据的类型分为：字节流和字符流

  • 字节流：以字节为单位，进行数据的读写

  • 字符流：以字符为单位，进行数据的读写

3.2 IO的流向说明

内存 ---输出---> 硬盘

硬盘 ---输入---> 内存

3.3 顶级分类

输入流 输出流
字节流 字节输入流：InputStream 字节输出流：OutputStream
字符流 字符输入流：Reader 字符输出流：Writer

四、字节流

4.1 字节输出流【OutputStream】

java.io.OutputStream抽象类是表示输出字节流的所有类的超类，将指定的字节(以字节为单位)信息写出到目的地。常用的子类为FileOutputStream

4.2 FileOutputStream类

java.io.FileOutputStream类是字节输出流，用于将数据写出到文件。

使用步骤：

  • 创建字节输出流对象，并绑定一个目的地文件
  • 向文件中写数据
  • 关闭流

4.2.1 构造方法

  • FileInputStream(File file)：参数要传递一个File类型的对象，表示向该文件中写数据，
  • FileOutputStream(String name)：参数需要传递一个字符串类型的文件路径，表示向该文件中写数据

4.2.2 其他方法

  • public abstract void write(int b)：向文件中写一个字节
  • public void write(byte[] b)：向文件中写一个字节数组的内容
  • public void write(byte[] b, int off, int len)：向文件中写入字节数据的一部分，参数off表示从哪个位置开始写，len表示写多少个字节。
  • public void close()：关闭流

使用流操作完成后，一定要调用close()方法关闭流。

4.2.4 追加写

FileOutputStream还有两个构造方法

  • FileOutputStream(File file, boolean append)：第二个参数append表示是否续写， true为续写,
  • FileOutputStream(String name, boolean append)：第二个参数append表示是否续写， true为续写。

这两个构造方法都需要一个boolean类型的参数，传递true表示在原来的基础上追加写数据。

使用这两个构造方法创建文件输出流对象，就可以指定是否追加写数据。

4.2.4 写换行

回车符\r和换行符\n：

  • 回车符：回到一行的开头
  • 换行符：下一行

不同系统中的换行：

  • Windows：每行的结尾是回车+换行，即\r\n
  • Unix：每行结尾只有换行，即\n
  • Mac：每行结尾是回车，即\r。从MacOS X开始于Unix统一。

4.3 字节输入流【InputStream】

java.io.InputStream抽象类是表示字节输入流的所有类的超类，可以读取字节信息到内存中。其中定义了字节输入流的一些共性方法：

  • public void close()：关闭流
  • int read()：从文件中读取一个字节并返回，如果文件已经读取结束了，返回的是-1
  • int read(byte[] b)：将数据读取到字节数组中，返回值是读取到的字节个数，如果文件已经读取结束了，返回-1。

4.4 FileInputStream类

java.io.FileInputStream类是文件输入流，从文件中读取字节。

4.4.1 构造方法

FileInputStream(File file)：参数需要传递一个File类型的文件，表示要从该文件中读取数据。

FileInputStream(String name)：参数需要传递一个字符串类型的文件路径，表示要从该文件中读取数据。

当你创建一个流对象时，必须传入一个文件路径。该路径下，如果没有该文件,会抛出FileNotFoundException(文件找不到异常)

五、字符流

当使用字节流读取文本文件时，可能会有一个小问题。就是遇到中文字符时，可能不会显示完整的字符，那是因为一个中文字符可能占用多个字节存储。所以Java提供一些字符流类，以字符为单位读写数据，专门用于处理文本文件。

5.1 字符输入流【Reader】

java.io.Reader抽象类是表示用于读取字符流的所有类的超类，可以读取字符信息到内存中。它定义了字符输入流的基本共性功能方法：

  • public void close()：关闭流。
  • public int read()：读取一个字符并返回，如果读取到末尾了，返回-1
  • public int read(char[] cbuf)：将数据读取到字符数组，并返回读取到的字符个数，读取到末尾返回-1

5.2 FileReader类

java.io.FileReader类是读取字符文件的便利类。构造时使用系统默认的字符编码和默认字节缓冲区。

5.2.1 构造方法

  • FileReader(File file)：参数需要传递File类型的文件对象，表示从该文件中读取数据
  • FileReader(String fileName)：参数需要穿覅字符串类型的文件路径，表示从该文件中读取数据

5.2.2 读取字符数据

读取字符：read()方法，每次可以读取一个字符的数据，提升为int类型，读取到文件末尾，返回-1

使用字符数组读取：read(char[] cbuf)，每次读取多个字符到数组中，返回读取到的有效字符个数，读取到末尾时，返回-1

5.3 字符输出流【Writer】

java.io.Writer抽象类是表示用于写出字符流的所有类的超类，将指定的字符信息写出到目的地。它定义了字节输出流的基本共性功能方法。

  • public abstract void close()：关闭此输出流并释放与此流相关联的任何系统资源。
  • public abstract void flush()：刷新此输出流并强制任何缓冲的输出字符被写出。
  • void write(int c)：写出一个字符。
  • write(char[] cbuf)：写字符数组
  • void write(char[] b, int off, int len)：写字符数组的一部分。off开始，写len个
  • void write(String str)：写出一个字符串。
  • void write(String str, int off, int len)：写字符串的一部分。off开始，写len个

5.4 FileWriter类

java.io.FileWriter类是写出字符到文件的便利类。构造时使用系统默认的字符编码和默认字节缓冲区。

5.4.1 构造方法

  • FileWriter(File file)：参数需要传递一个File类型的文件对象，表示向该文件写数据

  • FileWriter(String fileName)：参数需要传递一个字符串类型的文件路径，表示向该文件写数据

5.4.2 基本使用

步骤：

  1. 创建字符输出流
  2. 写数据
  3. 刷新
  4. 关闭流

只有字符输出流必须要刷新

5.4.3 关闭和刷新

因为内置缓冲区的原因，如果不关闭输出流，无法写出字符到文件中。但是关闭的流对象，是无法继续写出数据的。如果我们既想写出数据，又想继续使用流，就需要flush方法了。

  • flush()：刷新缓冲区，流对象可以继续使用。
  • close()：关闭流，释放系统资源。关闭前会刷新缓冲区。

注意：

  • 字符流，只能操作文本文件，不能操作图片，视频等非文本文件。
  • 当我们单纯读或者写文本文件时使用字符流其他情况使用字节流
