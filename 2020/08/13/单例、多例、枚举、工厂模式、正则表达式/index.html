一、单例模式

单例模式：是一种常用的软件设计模式。通过单例模式可以保证系统中，应用该模式的这个类只有一个
实例。即一个类只有一个对象实例。

1.1 实现步骤

  1. 将构造方法私有化，使其不能在类的外部通过new关键字实例化对象。
  2. 在类的内部产生一个唯一的一个实例化对象，并且将其封装为private static类型的成员变量
  3. 定义一个静态方法返回这个唯一对象

1.2 单例设计模式的类型

根据对象实例化的时机，单例模式可分为以下两种：

  • 饿汉单例模式
  • 懒汉单例模式

1.3 饿汉单例模式

饿汉单例模式：其含义就是在使用类的时候对象已经创建完毕，不管以后会不会使用到这个对象。

  • 单例模式可以保证系统中一个类只有一个实例对象。

二、多例模式

实例。多例类要自我创建并管理自己的实例，还要向外界提供获取本类实例的方法。

2.1 实现步骤

  4. 定义静态方法，给外界提供获取对象的方式。

五、正则表达式

5.1 正则表达式介绍

正则表达式：用来验证一个字符串是否符合规则。

在String中，有一个方法可以使用正则表达式验证。

  • boolean matches(String regex)：根据正则表达式验证字符串是否符合规则。

正则表达式的一些规则：

[abc]：表示a或者b或者c。
[^abc]：表示除了a,b,c之外的字符。
[a-z]：表示a-z中的任意一个。
[A-Z]：表示A-Z中的任意一个。
[0-9]：表示0-9中的任意一个。
[a-zA-Z0-9]：表示匹配a-z，A-Z，0-9中的任意一个字符
[a-mp-x]：表示匹配a-m或p-x中的任意一个字符
\d数字：[0-9]
\D 非数字：[^0-9]
\s空白字符：空格 tab等等
\S非空白字符
\w 单词字符：[a-zA-Z0-9_]
\W非单词字符：[^a-zA-Z0-9_]
?：0次或1次
*：任意次
+：一次或多次
{n}：恰巧n次
{n,}：至少n次
{n,m}：n到m次

正则有关的方法：

  • String[] split(String regex)：根据正则表达式切割字符串
  • String replaceAll(String regex, String replacement)：根据正则表达式进行替换
